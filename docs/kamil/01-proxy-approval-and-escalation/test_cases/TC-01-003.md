# TC-01-003: Take Action dialog opens with correct options

## Type
happy-path

## Preconditions
- An escalated submission exists on the dashboard
- User is logged in as admin

## Steps
1. Navigate to Dashboard > Escalations tab
2. Click "Take Action" on an escalated card

## Expected Result
- A modal dialog opens with title "Escalation: Approval timeout"
- Shows the escalation description (REQ ID, stage, absent approver)
- Shows submission summary (REQ ID, category badge, amount)
- Shows "Original Assignee" section with user avatar, name, role badge, and inactivity duration
- Two action buttons: "Proxy Approve" (green, "Approve on their behalf") and "Re-assign Approver" (blue, "Route to someone else")
- "View Submission Detail" link at the bottom

## Priority
critical
