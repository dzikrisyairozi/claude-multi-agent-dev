# TC-01-001: Cron detects and escalates overdue submissions

## Type
smoke

## Preconditions
- At least one submission exists with `status = 'pending'`, `date < NOW()`, `is_escalated = false`
- At least one admin/platform_admin user exists
- Cron endpoint is deployed and accessible

## Steps
1. Trigger the cron endpoint `GET /api/cron/check-escalations`
2. Query the database for the overdue submission

## Expected Result
- Submission's `is_escalated` is set to `true`
- Submission's `escalated_at` is set to the current timestamp
- A notification of type `escalation_timeout` is created for each admin/platform_admin user
- Notification has `requires_action = true`
- Submissions without a `date` field or with `date >= NOW()` are NOT escalated

## Priority
critical
