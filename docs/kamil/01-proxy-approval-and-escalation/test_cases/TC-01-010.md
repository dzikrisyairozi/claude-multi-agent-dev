# TC-01-010: Cron does not re-escalate already escalated submissions

## Type
edge-case

## Preconditions
- A submission is already escalated (`is_escalated = true`)
- The cron runs again

## Steps
1. Run the cron endpoint `GET /api/cron/check-escalations` a second time

## Expected Result
- The already-escalated submission is NOT processed again
- No duplicate notifications are created
- The cron only processes submissions where `is_escalated = false AND date < NOW() AND status = 'pending'`

## Priority
critical
