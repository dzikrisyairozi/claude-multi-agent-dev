# TC-01-005: Re-assign approver workflow completes successfully

## Type
happy-path

## Preconditions
- An escalated submission at step 2, originally assigned to "Erika"
- Multiple users with approver-compatible roles exist (e.g. Mita, Gilbert, Michael)
- User is logged in as admin

## Steps
1. Navigate to Dashboard > Escalations tab
2. Click "Take Action" on the escalated submission
3. Click "Re-assign Approver"
4. Review the original assignee info (Erika, "No action for 72h")
5. Select "Gilbert" from the list of available approvers
6. Click "Continue"

## Expected Result
- The current step's approver is updated to Gilbert
- `reassigned_from_user_id` is set to Erika's ID on the step
- `is_escalated` resets to `false`
- A `proxy_delegated` notification is sent to Gilbert with `requires_action = true`
- The escalation notification for the admin is marked as action completed
- The submission disappears from the Escalations tab
- The submission appears in Gilbert's pending approvals

## Priority
critical
